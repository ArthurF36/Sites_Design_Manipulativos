# 📚 Documentation_Overview

## 📘 Visão Geral

A documentação deste projeto foi organizada de forma modular para facilitar o entendimento, manutenção e uso dos diferentes componentes do código.

Em vez de concentrar todas as informações em um único arquivo, a documentação foi dividida em **três partes principais**, cada uma abordando um aspecto específico do projeto.

---

## 🧩 Estrutura da Documentação

### 1️⃣ Documentation_Python.md

Esta documentação é voltada para a execução de **scripts Python (`.py`)**.

**Inclui:**

* Como executar arquivos Python via terminal ou IDE
* Instalação de dependências
* Configuração do ambiente
* Solução de erros comuns relacionados à execução de scripts

👉 Ideal para quem deseja rodar o projeto de forma tradicional, sem uso de notebooks.

---

### 2️⃣ Documentation_Notebook.md

Focada no uso de **notebooks Jupyter (`.ipynb`)**, que são amplamente utilizados no projeto.

**Inclui:**

* Execução célula por célula
* Uso do botão **Run ▶**
* Configuração de kernel
* Execução da célula inicial de instalação (`pip install`)
* Boas práticas em ambientes interativos

👉 Indicada para análises exploratórias e execução interativa.

---

### 3️⃣ Documentation_Class_Logic

Esta documentação descreve a **lógica interna do projeto**, especialmente relacionada às classes e estruturas implementadas no projeto.

**Inclui:**

* Explicação das classes principais
* Responsabilidades de cada componente
* Fluxo de dados
* Organização da arquitetura do código

👉 Essencial para quem deseja entender ou modificar a lógica do sistema.

---

## 🎯 Objetivo da Separação

A divisão da documentação tem como objetivo:

* Melhorar a organização do projeto
* Facilitar a navegação das informações
* Atender diferentes perfis de uso (execução, análise, desenvolvimento)
* Tornar a manutenção mais simples e escalável

---

## ✅ Recomendação

Para melhor aproveitamento:

* Utilize **Documentation_Python.md** para execução via scripts
* Utilize **Documentation_Notebook.md** para uso interativo
* Consulte **Documentation_Class_Logic** para entender a estrutura interna de cada classe

---

🚀 Essa abordagem modular permite que você acesse rapidamente apenas a informação que precisa, sem sobrecarga de conteúdo.
